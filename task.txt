LIBRARY MANAGEMENT SYSTEM – PROJECT TASK DESCRIPTION

Project Name
Library Management System (C++ CLI Application)

Objective
The objective of this project is to build a simple command-line based library management system using C++.
The program allows users to manage books in a library by performing operations such as adding, viewing, borrowing, returning, searching, deleting, and updating books.
The system also stores book records in a file so that data is preserved even after the program exits.

---

1. INPUT (User Data)

When adding a book, the user must enter information about the book.

Example Input

Enter Book ID: 101
Enter Book Name: CleanCode
Enter Author Name: RobertMartin
Enter Quantity: 3

Required Inputs

• Book ID
• Book Name
• Author Name
• Quantity (Number of available copies)

These values are stored inside a Book object.

Example Book Class

class Book
{
public:
int id;
string name;
string author;
int quantity;
};

---

2. OUTPUT (Program Display)

Output is the information printed to the terminal.

Example Output

----- Book List -----

ID: 101
Name: CleanCode
Author: RobertMartin
Available: 3

The program reads stored book records and displays them to the user.

4. PROGRAM MENU

The application is menu driven.

Example Menu

1 Add Book
2 Show Books
3 Borrow Book
4 Return Book
5 Search Book
6 Delete Book
7 Update Book
8 Exit

User selects an option and the program performs the corresponding operation.

---

5. DATA STRUCTURE USED

The program stores books in a dynamic container.

vector<Book> library;

Every time a book is added:

library.push_back(book);

This allows the program to store an unlimited number of books during execution.

---

6. FILE STORAGE

Book records are saved in a file.

File Name

books_list.txt

Example File Content

101 CleanCode RobertMartin 3
102 CppPrimer Lippman 5

The program loads data from this file when it starts and saves updates back to the file.

---

↓
